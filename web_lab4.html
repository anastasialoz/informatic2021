Контрольні запитання

  1. Правило підключення стилів.
    За допомогою парного тегу style в блоці head або підключенням зовнішніх стилів в блоці head за допомогою тегу link з атрибутом rel зі значення stylesheet та атрибутом href в якому записується шлях до css-файлу (зазвичай відносний)
  2. Правила визначення селекторів стилів
    універсальний селектор "*" вибирає та надає стилі всім елементам. Приклад:p { color: blue; }
    селектор по типу елемента "div", "span", "nav"... вибирає тип елемента, до якого буде прикладене павило.
    селектор за класом ".menu", ".container", "btn" ... втибирає елементі в залежності від значення атрибута class. Приклад:.message { color: green;}
    селектор за ідентифікатором "#menu", "#container", "#btn" ... втибирає елементі в залежності від значення атрибута id. Приклад #introduction { color: red; }
  3. Пріоритети при визначенні стилів. Використання конструкції !important
    Для визначення який стиль буде мати елемент визначають специфічність селекторів: ідентифікатори вартують 100; классы вартують 10; селектори тега вартують 1. Селектор з найвищою специфічністю буде домінувати, незалежно від порядка, в якому появляються правила CSS. Якщо селекторы мають однакову специфічність, то пріорітетним буде той, що знаходиться нижче
    Змінити цей встановлений порядок пріоритетів можна з використанням конструкції !important. Приклад p { color: blue; ! }
  4. Інструмент розробника в браузері.
    Інструмент розробника - інструмент за замовчуванням у браузері Google Chrome та інших, який призначений для розробників та тестувальників з метою відладки коду. З його допомогою можна переглядати код сторінки, логи, завантаження сторінки, відображення в різних дозволах, помилки коду і тощо. Це спеціальний інструмент для перегляду стилів екземплярів об’єктів на сторінці браузера
  5. DOM модель гіпертекстового документа. Спадковість у стилізації.
    DOM модель гіпертекстового документа - це модель гіпертекстового документа,що полягає в його ієрархіній структурі, система елементів, вкладених одне в одного різних елементівта об’єктів.
    Принцип ієрархічного успадкування властивостей полягає в тому, що стилі, що назначено для батьківських об’єктів успадковуються вкладеними об’єктами всіх рівнів вкладеності. Ці стилі можна перевизначити в дочірніх об’єктах. Проте не всі стилі успадковуються
